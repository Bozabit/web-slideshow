  1. GIT

    Una introducción Básica

    1. GIT = VCS

      Version Control System que se refiere a un software que nos yuda a controlar las versiones de código de un sistema

    2. GIT desde 2005

      Fue fundado en el 2005 por Linus Torvalds, mientras trabaja con miles de personas en el desarrollo del Kernel de Linux

      Lo llamó: "the stupid content tracker"

    3. OTROS VCS => SVN, TFS

      Los sistemas de control de Cógido han existido desde 1986 por lo menos. Su éxito radica principalmente en los siquientes beneficios:

      1. Automatizacion del resgistro de cambios
      2. Branching & Merging
      3. Trazabilidad de cada línea

    4. Beneficios de GIT


      1. Uno de los más populares
      2. Es un software que corre local y tiene mucha portabilidad
      3. Tiene comunidades muy grandes de apoyo y herramientas como BitBucket y GitHub

  3. Una herramienta fundamental

    El manejo de GIT o en su defecto cualquier VCS, es una habilidad fundamental y básica para cualquier desarrollador.
